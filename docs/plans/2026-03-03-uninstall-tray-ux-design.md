# BackScreen 트레이 기본 동작 + 완전 삭제 언인스톨 디자인

## 배경
- 비개발자 사용자가 설치/제거를 스스로 수행할 수 있어야 한다.
- reference와 같이 "바탕화면에 항상 정보가 보이는" 체감을 위해 설정 창은 전면 노출보다 보조 도구여야 한다.
- 제거 시 사용자 데이터는 완전 삭제되어야 하며, 앱이 변경했던 바탕화면도 원복되어야 한다.

## 목표
- 앱 실행 시 기본 동작을 `트레이 상주 + 창 숨김`으로 전환한다.
- 앱 내부에서 `앱 제거` 버튼으로 언인스톨을 직접 실행한다.
- 제거 과정에서 사용자 데이터(`app.db`, `backups`, `wallpaper`)를 완전 삭제한다.
- 제거 전에 원본 배경화면으로 복구를 시도한다.

## 확정 요구사항
1. 비개발자가 앱 내부에서 쉽게 uninstall 가능해야 한다.
2. uninstall 시 사용자 데이터는 완전 삭제여야 한다.
3. 앱 내부 `앱 제거` 버튼은 언인스톨러를 직접 실행해야 한다.
4. 제거 전에 데이터 완전 삭제 경고 및 확인 입력(문구: `삭제`)이 필요하다.
5. 제거 시 배경화면 원복까지 포함한다.
6. 앱 기본 사용 경험은 `트레이 상주 + 필요 시만 설정창 열기`로 간다.

## 접근안 비교

### 접근안 A (채택): 앱 버튼 주도 + NSIS 훅 이중 안전장치
- 앱에서 확인 입력 검증 후:
  1) 배경 원복 시도
  2) 언인스톨러 실행
  3) 앱 종료
- NSIS `POSTUNINSTALL` 훅에서 앱 데이터 폴더 강제 삭제.
- 장점: 사용자 UX 단순, 데이터 삭제 보장성 높음.
- 단점: 앱 코드 + 설치 스크립트 수정 필요.

### 접근안 B: NSIS 훅 중심
- 앱 버튼은 언인스톨러 호출만 수행.
- 데이터 삭제/정리는 NSIS 단계에 집중.
- 장점: 앱 코드 변경 적음.
- 단점: 설치 스크립트 복잡도 증가, 디버깅 어려움.

### 접근안 C: 외부 helper 실행파일
- 별도 제거 도구가 앱 종료 대기/정리/삭제 수행.
- 장점: 책임 분리.
- 단점: 배포 복잡도 상승(YAGNI 위반 가능성 큼).

## 아키텍처 설계

### 1) 실행/노출 모델
- 앱 시작 시 메인 창을 즉시 숨기고 트레이 아이콘으로 상주.
- 트레이 메뉴:
  - 열기
  - 지금 배경 적용
  - 종료
- 닫기(X) 동작은 기존처럼 hide 유지.

### 2) 언인스톨 도메인
- Frontend:
  - 설정 패널 하단 `앱 제거(고급)` 카드 추가.
  - 경고 문구 + 확인 입력 `삭제` + 실행 버튼.
  - 진행 상태/실패 상태 메시지 표시.
- Rust command:
  - `uninstall_app(confirm_text)` 추가.
  - `confirm_text == "삭제"` 검증 실패 시 즉시 차단.
  - 원본 배경화면 복구 시도.
  - Windows uninstall 명령 조회/실행.
  - 성공 시 앱 종료.

### 3) 시스템 상태 저장
- 앱 데이터 디렉터리 하위에 `system_state.json` 추가.
- 최초 배경 적용 전에 현재 배경화면 경로를 1회 캡처.
- 예시 필드:
  - `originalWallpaperPath`
  - `capturedAt`

### 4) NSIS 훅
- `src-tauri/nsis-hooks.nsh` 추가.
- `NSIS_HOOK_POSTUNINSTALL`에서 앱 데이터 폴더 재귀 삭제.
- `tauri.conf.json`의 `bundle.windows.nsis.installerHooks` 연결.

## 데이터 흐름
1. 앱 최초 배경 적용 시: 시스템 배경 경로 캡처 -> `system_state.json` 저장.
2. 앱 제거 버튼 클릭 -> 경고/입력 검증 통과.
3. Rust가 배경 원복 시도 후 언인스톨러 실행.
4. 언인스톨 종료 후 NSIS 훅이 잔여 데이터 폴더 삭제.

## 오류 처리 원칙
- 입력 불일치: 제거 실행 불가(프론트와 백 모두 검증).
- 배경 원복 실패: 상태 메시지 표시 후 언인스톨은 진행(완전 삭제 우선).
- uninstall 명령 조회 실패: Windows `앱 및 기능`으로 이동 안내 제공.

## 테스트 전략
- 프론트 단위 테스트:
  - 확인 입력 없이는 버튼 비활성.
  - `삭제` 정확 일치 시에만 호출.
- 러스트 단위 테스트:
  - confirm 문자열 검증.
  - `system_state.json` 저장/로드 로직.
- 수동 E2E:
  - 설치 -> 배경 적용 -> 앱 제거 버튼 실행 -> `삭제` 입력 -> 제거 완료
  - 제거 후 데이터 폴더 미존재 확인
  - 제거 후 배경 원복 확인
