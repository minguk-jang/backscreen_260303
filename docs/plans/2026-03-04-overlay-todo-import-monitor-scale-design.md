# BackScreen Todo 위젯 + 급식 업로드 + 모니터/스케일 설정 디자인

- 작성일: 2026-03-04
- 범위: 스튜디오 UI, 바탕화면 렌더러, Tauri Windows 적용 레이어, 신규 인터랙션 위젯

## 승인된 결정

1. 급식 메뉴 아래에 Todo 리스트를 추가한다.
2. Todo는 체크 시 취소선 처리 후 맨 아래로 이동한다.
3. Todo 삭제는 스튜디오에서만 가능하다.
4. 바탕화면 인터랙션은 "정적 배경 + 별도 클릭 위젯" 구조로 구현한다.
5. 급식 업로드는 한글 문서 또는 PDF를 입력으로 받는다.
6. 기존 `이번 달 빠른 수정` 패널은 제거한다.
7. 모니터 대상은 `전체 / 주 모니터 / 특정 모니터 1개`를 지원한다.
8. 스튜디오에서 `콘텐츠 스케일(%)`만 조절한다(해상도 변경 아님).

## 목표

1. 비개발자 교사가 스튜디오에서 빠르게 수정/저장할 수 있어야 한다.
2. 배경은 기존처럼 안정적으로 유지하되, Todo는 바탕화면에서 즉시 체크할 수 있어야 한다.
3. 멀티 모니터 환경에서 표시 대상을 명확히 선택할 수 있어야 한다.
4. 텍스트/레이아웃 안정성을 유지하면서 콘텐츠 크기 조절이 가능해야 한다.

## 아키텍처

### 1) Studio(기존 메인 창)

- 데이터 입력/수정 중심 화면
- `이번 달 빠른 수정` 제거
- 추가 패널:
  - `Todo 관리` (추가/삭제/정렬 확인)
  - `디스플레이 설정` (모니터 대상 + 콘텐츠 스케일)
  - `급식 업로드` (한글/PDF 파일 선택 + 반영)

### 2) Wallpaper Renderer(기존 유지 + 확장)

- 정적 배경 이미지 생성 파이프라인은 유지
- `contentScalePercent`를 반영해 텍스트/패널 크기만 스케일
- 기존 overflow-safe 정책(자동 축소/말줄임/개수 제한) 유지

### 3) Interactive Todo Widget(신규)

- 바탕화면 위에 떠 있는 별도 Tauri 창(작은 고정형 위젯)
- 급식 요약 + Todo 체크박스 제공
- 체크 동작만 허용(삭제/편집 없음)
- 체크 즉시 순서 재정렬(미완료 상단, 완료 하단)

### 4) Windows 적용 레이어(확장)

- 배경 적용 시 모니터 대상 옵션 반영
- 모니터 목록 조회/선택 상태 저장
- 선택 불가 상태 발생 시 `주 모니터`로 자동 폴백

## 데이터 모델

```ts
interface TodoEntry {
  id: string;
  text: string;
  done: boolean;
  order: number;
  createdAt: string;
  doneAt?: string;
}

type MonitorTargetMode = "all" | "primary" | "single";

interface DisplaySettings {
  monitorMode: MonitorTargetMode;
  monitorId?: string;
  contentScalePercent: number; // 80~130
}

interface MealImportSettings {
  lastImportedAt?: string;
  lastSourceType?: "hwp" | "hwpx" | "pdf";
  lastFileName?: string;
}
```

- `AppState`에 `todos`, `display`, `mealImport`를 추가한다.
- 기본값:
  - `monitorMode: "primary"`
  - `contentScalePercent: 100`

## UX/플로우

### 스튜디오 편집 플로우

1. 급식 메뉴 패널에서 파일 업로드 또는 수동 입력
2. Todo 패널에서 항목 추가/삭제
3. 디스플레이 설정에서 모니터 대상과 스케일 조절
4. `저장 + 배경 적용` 실행

### 위젯 사용 플로우

1. 위젯에 오늘 급식과 Todo 노출
2. 사용자가 체크박스 클릭
3. 즉시 취소선 처리 + 목록 하단 이동
4. 상태 저장 후 다음 자동 적용/재실행에도 유지

## 급식 업로드 파싱 정책

1. 입력 파일 확장자: `hwp`, `hwpx`, `pdf`
2. 텍스트 추출 후 규칙 기반 파싱:
  - 날짜 토큰(예: `2026-03-04`, `3/4`, `3월 4일`) 탐지
  - 날짜별 메뉴 줄 묶기
3. 중복 날짜는 덮어쓰기(upsert)
4. OCR은 사용하지 않는다.
5. 파싱 실패 시 명확한 오류 안내와 수동입력 경로를 제공한다.

## 에러 처리

1. 파싱 실패: 파일명 + 실패 이유(날짜/메뉴 미검출) 표시
2. 모니터 선택 무효: 자동 `primary` 폴백 + 상태바 알림
3. 위젯 체크 저장 실패: UI 롤백 + 에러 안내
4. 스케일 범위 벗어남: 저장 전 검증 실패 처리
