# BackScreen 바탕화면 안전영역/텍스트 오버플로우 개선 디자인

- 작성일: 2026-03-03
- 범위: 배경 이미지 렌더링(`src/renderWallpaper.ts`)의 화면 배치/텍스트 가독성 개선
- 배경: 현재 배경이 전체 화면을 꽉 채우면서 비율 불일치와 텍스트 테두리 이탈이 발생함

## 1. 목표와 비목표

### 목표
1. 배경 콘텐츠를 전체 화면이 아닌 안전영역 내에 배치한다.
2. 좌측 12%, 상단 8% 영역은 비워서 레퍼런스와 같은 아이콘/여백 흐름을 유지한다.
3. 모든 텍스트는 카드 테두리 밖으로 절대 넘치지 않게 한다.
4. 텍스트 처리 우선순위는 `자동 축소 -> 말줄임(…)`으로 통일한다.

### 비목표
1. 데이터 모델(AppState) 구조 변경
2. 편집 UI(App.tsx)의 정보 구조 개편
3. 멀티 모니터 정책 변경

## 2. 사용자 결정 사항(승인)

1. 안전영역 기준: 비율 기반(`left=12%`, `top=8%`)
2. 긴 텍스트 기본 정책: `자동 축소`
3. 최소 폰트에서도 초과 시 정책: `말줄임(…)`
4. 접근안 선택: 안전영역 + 비율 레이아웃 + 자동축소(추천안)

## 3. 아키텍처 설계

기존 `renderWallpaperImage()`의 직접 좌표 계산 방식 대신, 먼저 레이아웃을 계산한 뒤 렌더링하는 2단계 구조로 전환한다.

1. `computeLayout(width, height)`
- 화면 크기에서 안전영역을 제외한 `contentRect`를 계산
- `contentRect` 내에 `headerRect`와 본문 3열(`columnRects`)을 생성
- 최소/최대 clamp를 통해 작은 해상도에서 왜곡 방지

2. `draw*` 함수 집합
- `drawHeaderSection`
- `drawTimetableSection`
- `drawMealsSection`
- `drawEventsSection`
- `drawPanelShell`

3. 텍스트 렌더 통합
- 텍스트는 공통 `fitText`를 통해서만 렌더링
- 직접 `ctx.fillText` 호출은 장식용 짧은 라벨로 제한

## 4. 텍스트 오버플로우 제어 규칙

1. 공통 규칙
- 모든 텍스트는 `fitText(ctx, text, { maxWidth, maxFont, minFont })`로 크기 축소를 시도
- `minFont`까지 내려간 뒤에도 초과 시 `truncateText(...)+…`

2. 패널별 규칙
- 시간표: 행 높이 고정, 표시 가능 행 수만 렌더
- 급식: 칩 높이 고정, 초과 항목은 `+N개`로 요약
- 일정: 카드 높이 고정, 상세는 1줄 한정 + 말줄임

3. 실패 방지
- 폭/높이가 비정상(0 이하)이면 빈 텍스트로 안전 반환
- 레이아웃이 극단적으로 작으면 제목만 유지하고 상세 생략

## 5. 데이터 흐름

1. 데이터 전처리
- `todaySlots`, `mealItems`, `monthlyEvents` 정렬/필터링
- 렌더 전 패널별 최대 후보 수 1차 제한

2. 레이아웃 계산
- `computeLayout`에서 전체 좌표를 먼저 확정

3. 렌더링
- 배경 -> 헤더 -> 3개 패널 순서로 그리기
- 각 패널은 내부 제약(폭/높이/행수)에 맞춰 항목 표시

## 6. 검증 계획

### 자동 테스트
1. `renderWallpaperImage`가 `1366x768`, `1920x1080`, `2560x1440`에서 예외 없이 data URL 반환
2. 매우 긴 텍스트 입력에서도 예외 없이 반환
3. `computeLayout`가 좌측/상단 비율(12%, 8%)을 계산하는지 검증

### 수동 테스트(Windows)
1. 좌측 아이콘 영역 미침범 확인
2. 상단 여백 8% 유지 확인
3. 긴 과목/급식/일정 텍스트의 카드 외부 이탈 여부 확인
4. 디스플레이 배율 100%/125%/150%에서 깨짐 여부 확인

## 7. 리스크와 대응
