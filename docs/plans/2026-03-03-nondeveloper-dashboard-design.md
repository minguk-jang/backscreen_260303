# BackScreen 비개발자 친화 대시보드 디자인

- 작성일: 2026-03-03
- 범위: 기존 단일 편집 화면을 비개발자 중심 UX로 재구성하고, 월간(일정/급식) 수정 효율을 극대화
- 배경: 현재 프로젝트는 `src/App.tsx` 단일 화면 기반으로 동작하며, 배경 렌더링은 `src/renderWallpaper.ts`, 저장/적용은 Tauri 명령(`src-tauri/src/lib.rs`)을 사용

## 1. 목표와 비목표

### 목표
1. 첫 설치 후 10분 내 설정 완료 가능
2. 화면 내 명확한 가이드라인 제공으로 입력 순서가 직관적일 것
3. 일정/급식(월 1회 이상 갱신 대상) 수정이 빠르고 쉬울 것
4. 기존 저장/적용 파이프라인(SQLite + 백업 + 배경 적용)은 유지

### 비목표
1. 다중 사용자/클라우드 동기화
2. 날씨 API 연동(현재 범위 제외)
3. 대규모 데이터 모델 개편

## 2. 사용자 우선순위 요약

- 최우선: 비개발자의 초기 설정 성공률
- 자주 수정: 일정, 급식 (매달 업데이트)
- 저빈도 수정: 시간표 (학기 단위)

## 3. 선택된 접근

선택안: **한 화면 대시보드 + 필수입력 체크리스트 + 월간 빠른수정 패널**

선정 이유:
1. 현재 구조를 크게 뒤엎지 않고 UX 개선 효과가 큼
2. 매달 수정이 필요한 일정/급식을 메인 작업영역으로 승격 가능
3. 시간표를 기본 접힘 처리하여 초기 인지부하를 낮출 수 있음

## 4. 정보 구조(IA)와 화면 설계

### 상단 고정 영역
1. 초기 설정 체크리스트 4단계
- 학교 정보 입력
- 이번 달 급식 1건 이상 등록
- 이번 달 일정 1건 이상 등록
- 배경 적용 1회 완료
2. 진행률 표시 (예: 2/4 완료)
3. 핵심 액션 버튼
- 저장
- 저장+배경 적용
- 지금 배경 적용

### 본문 2열 구조
1. 좌측: 이번 달 빠른 수정
- 일정 빠른 추가(날짜+제목 우선)
- 급식 빠른 추가(날짜+줄바꿈 입력)
- 이번 달 항목 우선 노출, 월 필터 제공
2. 우측: 기본 설정
- 학교 정보
- 시간표(기본 접힘 + "학기 1회 수정" 안내)
- 테마

### 상태/안내 영역
- 성공/실패 메시지에 다음 행동을 함께 제시
- 오류 발생 입력칸 시각 강조

## 6. 데이터 흐름 설계

1. 앱 진입
- `load_app_state` 호출 → `mergeWithDefaults` 적용 → 현재 월 기준 데이터 파생

2. 일정 빠른 수정
- 빠른 추가: 날짜+제목으로 즉시 행 생성
- 필요 시 인라인 확장으로 상세/색상 수정
- 저장 시 기존 저장 파이프라인 사용

3. 급식 빠른 수정
- 날짜 선택 + 줄바꿈 입력
- 동일 날짜 충돌 시 기본 동작은 덮어쓰기(옵션 확장 가능)

4. 시간표
- 기본 접힘
- 펼친 경우 기존 검증(시간 겹침/빈 과목) 유지

5. 배경 적용
- 저장+적용 원클릭 유지
- 성공 시 체크리스트 마지막 단계 완료 처리

## 7. 오류 처리 및 UX 문구 원칙

1. 기술 용어 최소화
- 나쁜 예: "JSON 파싱 실패"
- 좋은 예: "파일 형식이 맞지 않습니다. 다시 선택해 주세요."

2. 오류 메시지 규칙
- 문제 + 해결 행동을 함께 제공
- 상단 요약과 필드 강조를 동시에 제공

3. 안전장치
- 삭제 동작에 짧은 실행취소(undo) 제공
- 가져오기 실패 시 재시도 유도

## 8. 배경 결과물(레퍼런스 정렬)

- 유지: 상단 시간/현재수업 + 3패널(시간표/급식/학교활동)
- 개선: 텍스트 대비, 행 간격, 카드 밀도, 월간 일정 가독성
- 원칙: 편집 UI와 결과 배경의 역할을 분리해 편집 경험을 단순화

## 9. 검증 기준

1. 첫 실행 후 10분 내 체크리스트 4개 완료 가능
2. 일정 1건 수정 20초 이내(목표)
3. 급식 1일치 입력/수정 30초 이내(목표)
4. 저장/검증 실패 시 다음 행동이 메시지에 명확히 제시됨
